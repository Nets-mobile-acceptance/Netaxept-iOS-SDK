Swish m-commerce Integration

Swish Flow

To initiate Swish payment with Pia SDK, implement the SwishPaymentDelegate and trigger the process as follows:

/// Initiating Swish from a view controller that conforms to SwishPaymentDelegate.
/// The API returns false if Swish app cannot be opened, e.g. uninstalled or missing
/// URL scheme in info.plist of your application. 
/// - Note: The `sender` view controller is optionally passed to
///         display a transition view (an activity indicator)  
///         during the transition to and from Swish app (or bank ID app
///         opened by Swish).
guard PiaSDK.initiateSwish(fromSender: self, delegate: self) else {
    // Swish is not installed, preferably lead user to other payment methods
    return
}

Swish App Detection

The SDK checks if Swish can be launched as the first step in the process preventing unnecessary payment registration. If Swish app is not installed on user’s device, we recommend leading user to other available payment methods. The SDK throws wallet app not installed error in the corner case where user uninstalls Swish app after payment has been registered with merchant backend.

Note: In order for the PiA SDK to perform app detection, you need to add swish under the LSApplicationQueriesSchemes key in your Info.plist file. If you skip this step, the PiA SDK will always return wallet app cannot open error even though the Swish app is installed on the device. 

<key>LSApplicationQueriesSchemes</key>
<array>
    <string>swish</string>
</array>

Initializing a payment with Swish:

  • The SDK will notify the app when it needs to register the payment via SwishPaymentDelegate. The app should register payment with merchant backend and return the wallet URL in the completion handler.
  • Netaxpet requires a redirectUrl parameter which is the URL scheme of your application that is used to switch back to your application following app-switch to Swish app.

The following is an example of handling Swish payment registration:


// MARK: SwishPaymentDelegate - Swish Registration

    func registerSwishPayment(_ completionWithWalletURL: @escaping (String?) -> Void) {

        // Create a registration request body with order and payment details
        // The sample backend identifies Swish with its method ID:
        // `PaymentMethodID(id: "SwishM", displayName: "Swish", fee: 0)`
        let requestBody: RegistrationRequest // see section: Backend Models (1.1)

        URLSession.shared.makeRegisterRequest(with: requestBody) { transaction in
            guard let transaction = transaction else {
                return // handle error
            }

            // Grab the transaction ID, its used to later commit/rollback the registration
            // The SDK will callback the delegate with success/error result
            // Customer app is responsible for displaying the result to user
            self.transaction = transaction

            // Notify Pia SDK with registration result
            completionWithWalletURL(transaction.walletUrl)
        }
    }

Launching Swish App:

Once a payment registration succeeds with a transactionId and walletUrl, notify the SDK through registration completion handler passing the walletUrl.

After the transaction is performed in Swish (and bank ID) application, user is redirected to the merchant app via open(_ url:options:completionHandler) -> Bool method of UIApplication. The merchant app needs to notify the SDK using the API PiaSDK.applicationDidOpenFromRedirect(with: url, andOptions: options). 

func application(_ app: UIApplication, open url: URL, options: [UIApplicationOpenURLOptionsKey : Any] = [:]) -> Bool {

    // PiA SDK will return a boolean indicating if the redirect has been triggered by a payment app launched by PiA SDK.
    return PiaSDK.applicationDidOpenFromRedirect(with: url, andOptions: options)
}

The SDK will notify of successful redirect via swishDidRedirect delegate call. Merchant app should query payment status from Netaxept through merchant backend. The following code shows implementation of SwishPaymentDelegate in the sample app:


// MARK: SwishPaymentDelegate - Results

    func swishDidRedirect(_ transitionIndicatorView: UIView?) {
        // Query payment status and handle accordingly
    }

    func swishPaymentDidFail(with error: NPIError) {
        // Handle error
    }
