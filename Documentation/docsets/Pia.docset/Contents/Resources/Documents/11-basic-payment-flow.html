Basic payment flow description

Below you can find the interaction between the application that uses the PiA SDK and the SDK to start the basic payment flow by using the payment card:

In the basic payment flow, PiaSDKController should be instantiated with the following parameters:

  • merchantInfo - Uniquely identifies the merchant

    /**
 @param identifier Merchant id provided by Nets. Must not be nil.
*/
- (instancetype _Nonnull)initWithIdentifier:(NSString * _Nonnull)identifier;

/**
 @param identifier Merchant id provided by Nets. Must not be nil.
 @param testMode Default is false, set it to true to use Netaxept's test environment
*/
- (instancetype _Nonnull)initWithIdentifier:(NSString * _Nonnull)identifier testMode:(BOOL)testMode;

/**
 @param identifier Merchant id provided by Nets. Must not be nil.
 @param testMode Default is false, set it to true to use Netaxept's test environment
 @param cvcRequired Default is true, set it to false to make payments without Security Code
 Important: Note that disabling Security Code entry by setting _cvcRequired=False_ is submitted to both approval from your acquirer and activation by Netaxept customer service. Before proceeding, please contact your Netaxept customer service.
 */
- (instancetype _Nonnull)initWithIdentifier:(NSString * _Nonnull)identifier testMode:(BOOL)testMode cvcRequired:(BOOL)cvcRequired;

  • orderInfo - Describes the amount to be paid and the currency to be used

  • tokenCardInfo - Saved card token. This is optional - if nil, the user will have to input his card details

boolean testMode = true //Notify the SDK to use the test environment
let merchantInfo = NPIMerchantInfo(identifier: "your merchant ID here, provided by Nets", testMode)
let orderInfo = NPIOrderInfo(amount: NSNumber(value:4.5), currencyCode: "EUR")

let piaSDK = PiaSDKController(merchantInfo: merchantInfo, orderInfo: orderInfo, tokenCardInfo: nil)
piaSDK.piaDelegate = self
self.present(piaSDK, animated: true, completion: nil)

The PiaSDKController instance should also have a valid delegate. The SDK will notify the app when it needs to register the payment by calling doInitialAPICall. After the app registers the transaction, it should call the completionHandler received as a parameter to pass back the transactionInfo to PiA SDK.


// MARK: PiaSDKDelegate - Card registration 

// Delegate method called by the SDK for card payments and registration of card details (for later payments).
// The implementation below shows registration call to Nets test environment

/// Called after user has submitted **Card** details in SDK UI.
/// Obtain _transaction_ details from **Merchant** BE and call `completionHandler`
func doInitialAPICall(
    _ piaController: PiaSDKController,
    storeCard: Bool,
    withCompletion completionHandler: @escaping (NPITransactionInfo?) -> Void) {

    // Create a register request body with order and payment details
    let requestBody: RegistrationRequest // see: Backend Models (below)

    URLSession.shared.makeRegisterRequest(with: requestBody) { transaction in
        guard let transaction = transaction else {
            return // handle error
        }

        // The transaction ID is used to commit/rollback the registration
        // Pia SDK will callback the delegate with success/error result
        // Customer app is responsible for displaying the result to user
        self.transaction = transaction

        // notify Pia SDK with registration result
        completionHandler(NPITransactionInfo(
            transactionID: transaction.transactionId,
            redirectUrl: transaction.redirectOK)
        )
    }
}

// Sample implementation of registration request using `URLSession` 
// The models `RegistrationRequest` and `Transaction` can be found below under Backend Models

extension URLSession {
    func makeRegisterRequest(with requestBody: RegistrationRequest, callback: @escaping (Transaction?) -> Void) {
        /// Request to Net's (test) sample backend
        let urlRequest: URLRequest = {
            let merchantID = "Netaxept provided merchant ID"
            let baseURL = "https://api-gateway-pp.nets.eu/pia/test/merchantdemo/"
            var request = URLRequest(url: URL(string: baseURL + "v2/payment/\(merchantID)/register")!)
            request.httpMethod = "POST"
            let header = "application/json;charset=utf-8;version=2.0"
            ["Accept", "Content-Type"].forEach { headerField in
                request.setValue(header, forHTTPHeaderField: headerField)
            }

            // Encode order and payment details in the request
            request.httpBody = try! JSONEncoder().encode(requestBody)

            return request
        }()

        dataTask(with: urlRequest) { data, response, error in
            guard error == nil, let response = response as? HTTPURLResponse,
                200...299 ~= response.statusCode, let data = data else {
                    callback(nil)
                    return
            }

            // Decode registration result
            let transaction = try! JSONDecoder().decode(Transaction.self, from: data)
            DispatchQueue.main.async {
                callback(transaction)
            }
        }.resume()
    }
}

Note: Netaxept backend integration tips can be found in 5.4 Backend Tips, Register API section

Delegate

After the transaction has been registered, an unique transaction identifier transactionID will be returned from Netaxept to PiA SDK and PiA SDK will post it along with the card data to the Netaxept. In addition, an redirectUrl is required by PiA SDK so as that SDK can be notified by Netaxept after payment card data has been delivered to Netaxept successfully. During this phase, the 3DS authentication is performed as well unless it has been waived by your chosen card acquirer.

In the end, the PiaSDKController delegate will be notified when SDK handling is completed by getting a call to one of the following methods:

  • func piaSDK(_ PiaSDK: PiaSDKController, didFailWithError error: NPIError)

  • func piaSDKDidComplete(withSuccess PiaSDK: PiaSDKController)

  • func piaSDKDidCompleteSaveCard(withSuccess PiaSDK: PiaSDKController)

  • func piaSDKDidCancel(_ PiaSDK: PiaSDKController)

Registration Request Body

// Note: The payment method (`PaymentMethodID`) is `nil` for card payments 
//       and `PaymentMethodID(id: "EasyPayment", displayName: "Easy Payment", fee: 0)`
//       is used for payments with a previously stored stored card, aka easy-payment.
//       The `cardId` must have the card identifier token (String) for easy-payment 
//
struct RegistrationRequest: Encodable {
    let customerId: String
    let orderNumber: String
    let amount: Amount
    let storeCard: Bool?
    let method: PaymentMethodID?
    let cardId: String?             // set for payments with a stored card
    let items: LineItem?
    let paymentData: String?        // set for ApplePay payments
    let phoneNumber: String?        // set for Vipps payments
    let redirectUrl: String?        // set for Wallet App payments
    let customerEmail: String?      // set for Paytrail payments
    let customerFirstName: String?  // set for Paytrail payments
    let customerLastName: String?   // set for Paytrail payments
    let customerAddress1: String?   // set for Paytrail payments
    let customerPostCode: String?   // set for Paytrail payments
    let customerTown: String?       // set for Paytrail payments
    let customerCountry: String?    // set for Paytrail payments
}
