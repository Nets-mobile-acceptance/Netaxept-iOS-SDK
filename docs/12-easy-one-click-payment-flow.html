Easy (one-click) payment flow description

Use this flow to allow users to save their card information during initial payment and skip card detail entry next time they pay with your app again.

Note that for PSD2/SCA compliance reason, 3DSecure is activated and subsequent transactions can be made with or without security code entry (CVV/CVC) according to your preference.

Below you can find the interactions between the application that uses the PiA SDK and the SDK for the Easy Payment flow:

Initial transaction of Easy Payment flow

In the initial transaction of Easy Payment flow, PiA SDK should be started as similar as the basic payment flow. Then, Easy payment can be triggered if the user selects to save the card for the future usage during the card entry process. Then, SDK will pass the boolean parameter saveCard with true(by default, it is set to false to trigger the basic payment flow as mentioned above) in the register call to notify the app to go ahead with the Easy payment flow and you need to make the transaction registration according to the sequence diagram above.

After your app gets the successful result from PiA SDK, you can start Netaxept call(s) to proceed the transaction and fetch and store token in your backend which will be utilised for the subsequent transaction of Easy Payment.

Subsequent transaction of Easy Payment flow

In the subsequent transaction of Easy payment, you can follow the same way as the initial transaction to start PiA SDK and make sure:

  • you pass a non-nil tokenCardInfo when instantiating the PiA SDK
  • the register call from your backend contains the parameter recurringUse3DS = true in order to trigger 3DSecure authentication as per PSD2/SCA requirements.

Setting the security code (CVV/CVC):

You can set the boolean parameter cvcRequired on the TokenCardInfo instance to notify PiA SDK to ask for the CVV/CVC security code from the customer.

  • The register call from your backend contains the parameter recurringUse3DS = true in order to trigger 3DSecure authentication as per PSD2/SCA requirements.

The snippet below shows how to create an instance of NPITokenCardInfo:

let cardExpiryDate = "04/19"
let cvcRequired = true
let cardScheme: SchemeType = VISA
let tokenId = "492500******0004"
let cardTokenInfo = NPITokenCardInfo(tokenId: tokenId, schemeType: cardScheme, expiryDate: cardExpiryDate, cvcRequired: cvcRequired)

Important Note: If you wish to skip security code entry, you need approval from your acquirer.

The iOS SDK supports 3 different ways of presenting subsequent payments to the user:

  • Option 1: Require user confirmation: Present a card view with entry field for CVC (if CVC is required) and a “Pay XX.XX” button to confirm payment
  • Option 2: Skip user confirmation - with SDK UI: Present a card view with an activity indicator while processing payment without requiring user confirmation
  • Option 3: Skip user confirmation - without SDK UI: Process payment staying in customer’s view controller (presenting 3DS authentication webpage on the customer’s controller if necessary)

Note: Options 2 and 3 are not applicable if CVC is required. Approval must be granted by your acquirer.

Option 1 Option 2 Option 3

The first two options can be presented using the following API passing the appropriate value for requireCardConfirmation parameter.

let piaController = PiaSDKController(
    testMode: testMode,
    tokenCardInfo: cardInfo,
    merchantID: merchantID,
    orderInfo: orderInfo,
    requireCardConfirmation: requireCardConfirmation
)

present(piaController, animated: animated, completion: completion)

Skip confirmation - without SDK UI: The following code snippet demonstrates this option. This is also the preferred flow used in the Pia sample app. 

PiaSDK.showActivityIndicator(in: sender) // `sender` is customer's view controller

// Register payment with your merchant backend and obtain a `transactionID`
api.registerCardPay(orderDetails: orderDetails) { result in 

    let success = {
        // Typically commit the transaction (with merchant backend) and show result to user
    }

    let failure: (NPIError) -> Void = { error in
        // Handle failure
    }

    let cancellation = {
        // Handle cancellation
    }

    // Note: Make sure to call this only from main queue
    // The SDK could present 3DS authentication on `sender`
    PiaSDK.initiateTokenizedCardPay(
        from: sender,
        testMode: testMode,
        showsActivityIndicator: true,
        merchantID: merchantID,
        redirectURL: result.redirectURL,
        transactionID: result.transactionID,
        success: success,
        cancellation: cancellation,
        failure: failure
    )
}

Note: Netaxept backend integration tips can be found in 5.4 Backend Tips, Registering Easy Payment section
